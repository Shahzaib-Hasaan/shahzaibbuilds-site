## 🔴 CRITICAL (Killing conversions right now)

### 1. Split Calendly URLs — Active Lead Leakage
**Files:** `components/portfolio/HeroSection.tsx:29`, `components/portfolio/StickyCTA.tsx:40`, `components/portfolio/ServicesSection.tsx:46` vs `lib/emailCapture.ts:99`

The **email-gated CTA buttons** (Hero, StickyCTA, ServicesSection) hardcode:
```
https://calendly.com/shahxeebhassan/30min
```
But `getCalendlyUrl()` in `emailCapture.ts` (the function that should be the source of truth) uses:
```
https://calendly.com/shahzaibbuilds/30min
```
These are **two different Calendly profiles** — different usernames. A founder who was already captured and returns directly (skipping the modal) gets sent to the personal-name URL. A first-time visitor going through email capture gets the branded URL. Both may have different calendar availability or appearance.

**Why it kills conversions:** Even a 1-second "huh, that's odd" thought at the booking page destroys trust. You're fragmenting your funnel and the inconsistency signals sloppiness to an already-skeptical technical audience.

## 🟢 GOOD (What's actually working well)

1. **Email gate architecture is correct** — `isEmailCaptured()` in localStorage with 24h expiry, pre-filling Calendly with `?email=`, and the n8n webhook pipeline is a professional-grade conversion flow. Most solo operators don't build this level of funnel infrastructure.

2. **`getCalendlyUrl(email?)` utility in `emailCapture.ts`** — centralizing the Calendly URL in one function is the right pattern. The bug is that components don't use it; they hardcode the URL instead.

3. **SEO fundamentals are strong** — `app/layout.tsx` has proper OG tags, Twitter card, canonical URL, and three schema types (ProfessionalService, Person, FAQPage). This is well above average for a solo operator site.

4. **Glass card component system** — `.glass-card` custom class and the dark theme are visually distinctive and appropriate for the AI/tech persona. Not generic.

5. **StickyCTA dismiss behavior** — the dismiss resets when scrolling back to top (`setIsDismissed(false)` when `scrollY < windowHeight * 0.3`), which is smart. Users who scroll up are re-engaged.

6. **Chatbot response parsing handles multiple n8n output shapes** — the chatbot tries `string | array | object` response shapes, which is resilient for a webhook-based backend that may return varied formats.

7. **Framer Motion `viewport={{ once: true }}`** — scroll animations fire only once, not on every scroll-into-view. Correct performance pattern.

8. **Mobile tap targets on StickyCTA** — `min-h-[44px] min-w-[44px]` on dismiss button shows attention to WCAG tap target requirements.

9. **`priority` on hero image** — `<Image priority>` on the above-fold profile photo is correct Next.js LCP optimization.

10. **`getCalendlyUrl` pre-fills email** — passing `?email=` to Calendly removes friction at the booking step. Smart.
